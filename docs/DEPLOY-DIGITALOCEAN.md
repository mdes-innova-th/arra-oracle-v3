# Deploy Arra Oracle to DigitalOcean

This runbook is the operator path for a small public Arra HTTP node on
DigitalOcean using the published GHCR HTTP image.

It covers provision, firewall, optional app token, client connection, seeding,
and teardown. The default target is a short-lived test droplet in `sgp1` using
`s-1vcpu-2gb` (about USD $12/month if left running).

## Prerequisites

- DigitalOcean account with billing enabled.
- [`doctl`](https://docs.digitalocean.com/reference/doctl/) installed and authenticated.
- A DigitalOcean API token with droplet/firewall permissions.
- An SSH key uploaded to DigitalOcean.
- Docker image published to GHCR:
  `ghcr.io/soul-brews-studio/arra-oracle-v3:http`.
- Local Arra CLI if you want to test from the shell:
  `bunx --bun arra@github:Soul-Brews-Studio/arra-oracle-v3 --help`.

Authenticate `doctl`:

```bash
doctl auth init
# or, in CI/non-interactive shells:
export DIGITALOCEAN_ACCESS_TOKEN=dop_v1_...
doctl account get
```

Find your SSH key id:

```bash
doctl compute ssh-key list
```

Verify the image exists and is multi-arch:

```bash
docker buildx imagetools inspect ghcr.io/soul-brews-studio/arra-oracle-v3:http
```

## Provision with the deploy script

The intended operator wrapper is `scripts/deploy-do.sh`. It should create a
small droplet, install Docker, run the GHCR HTTP image, attach a firewall, and
print the droplet IP plus cleanup command.

Typical usage:

```bash
export DIGITALOCEAN_ACCESS_TOKEN=dop_v1_...
export DO_SSH_KEY_ID=<ssh-key-id-or-fingerprint>
export ARRA_API_TOKEN=$(openssl rand -hex 24)

scripts/deploy-do.sh \
  --name arra-m5-test \
  --region sgp1 \
  --size s-1vcpu-2gb \
  --image ghcr.io/soul-brews-studio/arra-oracle-v3:http \
  --ssh-key "$DO_SSH_KEY_ID" \
  --allow-ssh-from "$(curl -fsSL https://api.ipify.org)/32" \
  --allow-http-from "$(curl -fsSL https://api.ipify.org)/32" \
  --token "$ARRA_API_TOKEN"
```

Expected outputs to save:

- droplet id
- public IPv4 address
- firewall id
- `ARRA_API_TOKEN` value, if generated by the script
- teardown command using `scripts/destroy-do.sh`

### Manual fallback with doctl

Use this if the wrapper script is unavailable or you need to inspect each step.
Replace the SSH key id/fingerprint and allowlist CIDRs first.

```bash
export DROPLET_NAME=arra-m5-test
export DO_REGION=sgp1
export DO_SIZE=s-1vcpu-2gb
export DO_SSH_KEY_ID=<ssh-key-id-or-fingerprint>
export ALLOW_SSH_FROM="$(curl -fsSL https://api.ipify.org)/32"
export ALLOW_HTTP_FROM="$ALLOW_SSH_FROM"
export ARRA_API_TOKEN=$(openssl rand -hex 24)

cat > /tmp/arra-do-cloud-init.yaml <<'YAML'
#cloud-config
package_update: true
packages:
  - ca-certificates
  - curl
runcmd:
  - curl -fsSL https://get.docker.com | sh
  - mkdir -p /opt/arra/data
  - docker pull ghcr.io/soul-brews-studio/arra-oracle-v3:http
  - docker rm -f arra-oracle || true
  - >-
    docker run -d --name arra-oracle --restart unless-stopped
    -p 47778:47778
    -v /opt/arra/data:/data
    -e ORACLE_DATA_DIR=/data
    -e ORACLE_PORT=47778
    -e PORT=47778
    -e ARRA_API_TOKEN=${ARRA_API_TOKEN}
    ghcr.io/soul-brews-studio/arra-oracle-v3:http
YAML
# Substitute local token into cloud-init before creating the droplet.
perl -0pi -e "s/\$\{ARRA_API_TOKEN\}/$ARRA_API_TOKEN/g" /tmp/arra-do-cloud-init.yaml

doctl compute droplet create "$DROPLET_NAME" \
  --region "$DO_REGION" \
  --size "$DO_SIZE" \
  --image ubuntu-24-04-x64 \
  --ssh-keys "$DO_SSH_KEY_ID" \
  --user-data-file /tmp/arra-do-cloud-init.yaml \
  --wait

DROPLET_ID=$(doctl compute droplet list --format ID,Name --no-header | awk -v n="$DROPLET_NAME" '$2==n {print $1; exit}')
DROPLET_IP=$(doctl compute droplet get "$DROPLET_ID" --format PublicIPv4 --no-header)
printf 'DROPLET_ID=%s\nDROPLET_IP=%s\n' "$DROPLET_ID" "$DROPLET_IP"
```

Create and attach a firewall:

```bash
doctl compute firewall create \
  --name "${DROPLET_NAME}-fw" \
  --droplet-ids "$DROPLET_ID" \
  --inbound-rules "protocol:tcp,ports:22,sources:${ALLOW_SSH_FROM}" \
  --inbound-rules "protocol:tcp,ports:47778,sources:${ALLOW_HTTP_FROM}" \
  --outbound-rules "protocol:tcp,ports:all,destinations:0.0.0.0/0" \
  --outbound-rules "protocol:udp,ports:all,destinations:0.0.0.0/0"
```

Wait for Docker to start the container:

```bash
ssh root@"$DROPLET_IP" 'docker ps && docker logs --tail=80 arra-oracle'
curl -sf "http://$DROPLET_IP:47778/api/health"
```

## Security model

Use two layers:

1. **Firewall/network allowlist.** Restrict SSH (`22`) and Arra HTTP (`47778`)
   to your office/home/VPN CIDR. This is the primary protection for a temporary
   public droplet.
2. **`ARRA_API_TOKEN` app guard, opt-in.** The deploy wrapper may configure a
   bearer-token guard for non-health API paths. Send it as
   `Authorization: Bearer <token>` when calling protected endpoints.

`/api/health` should stay open so DigitalOcean, scripts, and humans can verify
liveness without a secret. Local MCP stdio mode is unaffected by this remote
HTTP token because stdio runs locally and does not expose a public HTTP port.

If the current source build you deploy does not yet enforce `ARRA_API_TOKEN`,
keep the firewall allowlist tight and treat the token as forward-compatible
configuration for the deploy wrapper/app guard.

## Connect a client

Set the HTTP target:

```bash
export ORACLE_API="http://$DROPLET_IP:47778"
```

Basic health check:

```bash
arra health
# or with the package directly:
bunx --bun arra@github:Soul-Brews-Studio/arra-oracle-v3 health
```

For one-off named targets, store or select a target with the operator CLI config
flow, then use `--at`:

```bash
arra config set target.m5.url "http://$DROPLET_IP:47778"
arra --at m5 health
arra --at m5 stats
arra --at m5 search "nothing deleted"
```

When token auth is enabled, pass the bearer token for protected HTTP calls:

```bash
curl -sf -H "Authorization: Bearer $ARRA_API_TOKEN" \
  "http://$DROPLET_IP:47778/api/stats"

curl -sf -H "Authorization: Bearer $ARRA_API_TOKEN" \
  "http://$DROPLET_IP:47778/api/search?q=nothing+deleted"
```

If your CLI build does not yet attach `ARRA_API_TOKEN` automatically, use the
firewall allowlist, an SSH tunnel, or `curl` for protected-token checks:

```bash
ssh -L 47778:127.0.0.1:47778 root@"$DROPLET_IP"
ORACLE_API=http://127.0.0.1:47778 arra health
```

## Seed data

Choose one path.

### Option A: fresh index on the droplet

Use this for clean tests or demos. Copy/clone the source you want indexed, then
run the indexer against it inside a one-off container or from a source checkout.

```bash
ssh root@"$DROPLET_IP" 'mkdir -p /opt/arra/source'
rsync -az --delete ./ψ root@"$DROPLET_IP":/opt/arra/source/ 2>/dev/null || true

ssh root@"$DROPLET_IP" <<'REMOTE'
docker run --rm \
  -v /opt/arra/data:/data \
  -v /opt/arra/source:/source:ro \
  -e ORACLE_DATA_DIR=/data \
  -e ORACLE_REPO_ROOT=/source \
  ghcr.io/soul-brews-studio/arra-oracle-v3:http \
  bun run index
REMOTE
```

### Option B: ship the m5 index

Use this when you want the droplet to mirror an already-built m5 data directory.
Stop the container, copy the data, and restart it:

```bash
ssh root@"$DROPLET_IP" 'docker stop arra-oracle'
rsync -az ~/.oracle/ root@"$DROPLET_IP":/opt/arra/data/
ssh root@"$DROPLET_IP" 'docker start arra-oracle'
```

If a dedicated seed/export helper lands in this repo, prefer it and link it here
from the deploy wrapper. Until then, the safe contract is: preserve the contents
of `/opt/arra/data` because that is the Docker volume backing `$ORACLE_DATA_DIR`.

## Teardown

Destroy test droplets when done to stop the monthly cost.

```bash
scripts/destroy-do.sh --name arra-m5-test
```

Manual fallback:

```bash
DROPLET_ID=$(doctl compute droplet list --format ID,Name --no-header | awk '$2=="arra-m5-test" {print $1; exit}')
FW_ID=$(doctl compute firewall list --format ID,Name --no-header | awk '$2=="arra-m5-test-fw" {print $1; exit}')

[ -n "$FW_ID" ] && doctl compute firewall delete "$FW_ID" --force
[ -n "$DROPLET_ID" ] && doctl compute droplet delete "$DROPLET_ID" --force
```

Check nothing remains:

```bash
doctl compute droplet list --format ID,Name,PublicIPv4,Status
doctl compute firewall list --format ID,Name
```

## Troubleshooting

- **`curl /api/health` fails:** check the firewall allowlist, droplet IP, and
  `ssh root@IP 'docker ps && docker logs arra-oracle'`.
- **SSH works but HTTP does not:** confirm port `47778` is allowed in the DO
  firewall and the container was started with `-p 47778:47778`.
- **Protected API returns unauthorized:** resend
  `Authorization: Bearer $ARRA_API_TOKEN`, or verify the deploy wrapper wrote
  the same token into the running service.
- **CLI points at localhost:** set `ORACLE_API=http://<ip>:47778` or use
  `arra --at <name> ...` after configuring the named target.
