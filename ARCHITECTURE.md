## 1. Tech Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Bun >= 1.2.0 (README), >= 1.3.0 recommended (LOCAL-DEV) |
| HTTP API | Elysia + `@elysiajs/cors` + `@elysiajs/swagger` |
| MCP | `@modelcontextprotocol/sdk` stdio server |
| ORM | Drizzle ORM with `bun:sqlite` |
| Search | SQLite FTS5 + LanceDB vector search (hybrid) |
| CLI | `commander` in `cli/` and `src/vault/cli.ts` |
| Web UI | Astro project in `web/` (separate dev server) |
| Language | TypeScript 5.7 |

## 5. Database Schema (Drizzle)

### Core tables

| Table | Purpose |
|-------|---------|
| `oracle_documents` | Indexed markdown metadata (id, type, source_file, concepts, supersededBy, origin, project) |
| `indexingStatus` | Singleton progress tracking |
| `indexingJobs` | Per-doc per-model vector embedding queue |
| `searchLog` | Query audit log |
| `consultLog` | Legacy consultation history (retained) |
| `learnLog` | Learning/pattern audit |
| `documentAccess` | Document access audit |

### FTS5

Virtual table `oracle_fts` for full-text search over document content + concepts.

## 6. Hybrid Search

1. Sanitize query (strip FTS5 special chars)
2. Run SQLite FTS5 keyword search
3. Run LanceDB vector nearest-neighbor search
4. Normalize scores:
   - FTS5: `e^(-0.3 * |rank|)`
   - Vector: `1 - distance`
5. Merge + deduplicate by document ID
6. Score: 50% FTS + 50% vector + 10% boost if in both
7. Return results with metadata

Graceful degradation: if vector store unavailable, return FTS5-only with `vectorAvailable: false`.

## 7. Configuration

### Environment variables (from `src/config.ts` + README)

| Variable | Default | Purpose |
|----------|---------|---------|
| `ORACLE_PORT` | `47778` | HTTP server port |
| `ORACLE_DATA_DIR` | `~/.arra-oracle-v2` | SQLite DB, vectors, plugins, logs |
| `ORACLE_DB_PATH` | `$ORACLE_DATA_DIR/oracle.db` | SQLite path |
| `ORACLE_REPO_ROOT` | auto | Knowledge base root where `ψ/` lives |
| `VECTOR_URL` | — | Proxy vector requests to this base URL |
| `VECTOR_FALLBACK` | `fts5` | Behavior when vector proxy unreachable |
| `CORS_ORIGIN` | — | Allowed CORS origin for web app |

## 8. Plugin System

- Builtin plugins: `src/server/plugin/builtin.ts`
- Loader: `src/server/plugin/loader.ts`
- Registry: `src/server/plugin/registry.ts`
- External plugins can be dropped into `PLUGINS_DIR` (under data dir)
- Env filtering: `ORACLE_ENABLED_PLUGINS`, `ORACLE_DISABLED_PLUGINS`

---

## 9. Key Integration Points

- **Jit / innova-bot**: `scripts/bootstrap.sh` starts Arra Oracle at `ORACLE_PORT=47778` for the multi-agent system
- **MCP clients**: Claude Code, Codex must point at same `ORACLE_DATA_DIR`
- **Web app**: `web/` Astro dashboard talks to HTTP API via `PUBLIC_BACKEND_URL`
- **Vault sync**: `oracle-vault` pushes/pulls `ψ/` content to/from GitHub

---

## 10. Notes for Operators

- Fresh install starts with empty index; run indexer or use `oracle_learn` MCP tool to populate
- Backend does not require LanceDB to function; FTS5 works standalone
- Data directory default still uses legacy name `.arra-oracle-v2`
- Plugin autoload from data directory can execute code — review plugins before enabling
